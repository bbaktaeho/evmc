# Project Guide

## Tech Stack

- Go 1.24+
- go-ethereum `rpc` package (transport layer)
- Supported chains: Ethereum, Kaia (and any EVM-compatible chain)

## Project Structure

```
evmc.go              - Evmc struct, single entry point
procedure.go         - RPC method name constants (Procedure type)
*_namespace.go       - Namespace implementations (eth, web3, debug, kaia)
contract.go          - Raw contract call helpers
evmctypes/           - Core EVM types and JSON unmarshaling
  evmctypes.go       - Type definitions: Block, Transaction, Receipt, Log, …
  *_unmarshaling.go  - Custom UnmarshalJSON for hex-encoded fields
evmcsoltypes/        - Solidity ABI types
evmcutils/           - Utility helpers
erc20.go             - Auto-generated by abigen (DO NOT EDIT)
examples/            - Usage examples per namespace
testdata/mainnet/    - Golden test JSON fixtures
```

## Architecture

`Evmc` is the single entry point. All namespaces are accessed via getters:

```
client.Eth()          -> *ethNamespace    (eth_*)
client.Web3()         -> *web3Namespace   (web3_*)
client.Debug()        -> *debugNamespace  (debug_*)
client.Kaia()         -> *kaiaNamespace   (kaia_*)
client.Contract()     -> *contract        (raw call/call)
client.ERC20(addr)    -> ERC-20 token helpers
client.ERC721(addr)   -> ERC-721 token helpers
client.ERC1155(addr)  -> ERC-1155 token helpers
```

## Key Files

| File | Role |
|------|------|
| `procedure.go` | All RPC method name constants as `Procedure` type. Add here first before implementing any method. |
| `evmctypes/evmctypes.go` | Shared response types. Define new types here. |
| `evmctypes/*_unmarshaling.go` | `UnmarshalJSON` implementations for hex fields. One file per namespace or type group. |
| `erc20.go` | Auto-generated — never edit manually. |

## Code Conventions

- Receiver: first letter of struct name (`e` for `ethNamespace`)
- Error wrap: `fmt.Errorf("MethodName: %w", err)`
- Context: accept `context.Context` in public API; use `context.Background()` internally
- Hex fields: decode in `*_unmarshaling.go`, not inline
